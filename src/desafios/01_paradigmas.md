## 🔨 O Desafio do Criador

### Missão: As Primeiras Formas do Caos e da Ordem

**Contexto narrativo:**

O Caos Primordial reina no vazio digital. Você, jovem Deus Criador, deve demonstrar que compreende tanto o mundo caótico (estruturado) quanto o mundo ordenado (orientado a objetos). Sua missão é provar que sabe pensar nos dois paradigmas — para então escolher o caminho da criação.

**Sua missão como Deus Criador:**

1. No mundo do Caos (estruturado), crie uma `struct` para representar um **Guerreiro** com `nome`, `vida` e `forca`. Crie funções separadas para `atacar` (reduz a vida do inimigo pela força do atacante) e `exibirStatus`.

2. No mundo da Ordem (orientado a objetos), crie uma classe `Guerreiro` em Java com os mesmos atributos e métodos — mas agora **dentro** da classe.
   1. No `main`, crie **dois guerreiros** e faça um atacar o outro. Exiba o status antes e depois do ataque.

**Requisitos técnicos obrigatórios:**

- [ ] Versão em C com `struct` e funções separadas
- [ ] Versão em Java com classe e métodos integrados
- [ ] Dois objetos/instâncias de guerreiro em cada versão
- [ ] Um guerreiro ataca o outro (a vida do alvo diminui)
- [ ] Exibição do status antes e depois do ataque

**Critérios de avaliação:**

| Critério                            | Pontos      |
| ----------------------------------- | ----------- |
| Struct e funções corretas em C      | 20 pts      |
| Classe Java com atributos e métodos | 30 pts      |
| Criação de dois guerreiros          | 15 pts      |
| Lógica de ataque implementada       | 20 pts      |
| Exibição de status funcional        | 15 pts      |
| **Total**                           | **100 pts** |

**Dica do Oráculo:** _"Observe como, no Java, o guerreiro sabe atacar por conta própria. Em C, alguém precisa dizer a ele o que fazer. Essa diferença parece sutil agora... mas ela mudará tudo."_

## 🗣️ Palavras de inspiração

> _"Na programação estruturada, você dá ordens ao computador. Na orientação a objetos, você cria seres que sabem se virar sozinhos."_

> _"Uma struct é um formulário. Uma classe é uma criatura. O formulário não faz nada — a criatura age."_

> _"O paradigma que você escolhe não muda só o código. Muda a forma como você enxerga o problema."_

> _"Na programação estruturada, os dados são como órgãos espalhados pelo chão. Na OO, eles ganham um corpo, uma armadura e uma vontade própria."_

> _"Antes de criar o universo, decida suas leis. Esse é o poder de escolher um paradigma."_
